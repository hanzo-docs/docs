The VMs API lets you programmatically manage virtual machines across all connected cloud providers.

## List VMs

```bash
GET /v1/vms
```

Query parameters:

| Param | Type | Description |
|-------|------|------------|
| `provider` | string | Filter by provider (`aws`, `digitalocean`, `hetzner`) |
| `region` | string | Filter by region |
| `status` | string | Filter by status (`running`, `stopped`, `provisioning`) |
| `limit` | integer | Max results (default 50, max 200) |
| `cursor` | string | Pagination cursor |

Example:

```bash
curl -X GET "https://api.visor.hanzo.ai/v1/vms?provider=digitalocean&status=running" \
  -H "Authorization: Bearer $VISOR_API_KEY"
```

Response:

## Create a VM

```bash
POST /v1/vms
```

Request body:

| Field | Type | Required | Description |
|-------|------|----------|------------|
| `name` | string | yes | VM name (alphanumeric, hyphens) |
| `provider` | string | yes | Cloud provider |
| `region` | string | yes | Provider region |
| `size` | string | yes | Plan size |
| `image` | string | yes | OS image slug |
| `ssh_keys` | string[] | no | SSH key IDs to inject |
| `tags` | object | no | Key-value tags |
| `user_data` | string | no | Cloud-init script |
| `provider_options` | object | no | Provider-specific options |

Example:

```bash
curl -X POST https://api.visor.hanzo.ai/v1/vms \
  -H "Authorization: Bearer $VISOR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "api-server-02",
    "provider": "aws",
    "region": "us-east-1",
    "size": "standard",
    "image": "ubuntu-24.04",
    "ssh_keys": ["key_abc123"],
    "tags": {
      "env": "staging"
    }
  }'
```

## Get a VM

```bash
GET /v1/vms/:id
```

Returns full VM details including monitoring metrics.

## Update a VM

```bash
PATCH /v1/vms/:id
```

Updatable fields:

| Field | Description |
|-------|------------|
| `name` | Rename the VM |
| `size` | Resize (may require restart) |
| `tags` | Update tags |

```bash
curl -X PATCH https://api.visor.hanzo.ai/v1/vms/vm_abc123 \
  -H "Authorization: Bearer $VISOR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "size": "performance",
    "tags": { "env": "production" }
  }'
```

## Delete a VM

```bash
DELETE /v1/vms/:id
```

This permanently destroys the VM and its boot disk. Attached volumes are detached but not deleted.

```bash
curl -X DELETE https://api.visor.hanzo.ai/v1/vms/vm_abc123 \
  -H "Authorization: Bearer $VISOR_API_KEY"
```

## VM actions

### Start

```bash
POST /v1/vms/:id/actions/start
```

### Stop

```bash
POST /v1/vms/:id/actions/stop
```

### Restart

```bash
POST /v1/vms/:id/actions/restart
```

### Rebuild

Reinstall the OS image. This wipes the boot disk.

```bash
POST /v1/vms/:id/actions/rebuild
```

```json
{
  "image": "ubuntu-24.04"
}
```

## Available images

```bash
GET /v1/images
```

Common images:

| Slug | OS |
|------|----|
| `ubuntu-24.04` | Ubuntu 24.04 LTS |
| `ubuntu-22.04` | Ubuntu 22.04 LTS |
| `debian-12` | Debian 12 Bookworm |
| `fedora-41` | Fedora 41 |
| `alpine-3.21` | Alpine Linux 3.21 |
| `rocky-9` | Rocky Linux 9 |
| `windows-2022` | Windows Server 2022 |

## Webhook events

Subscribe to VM lifecycle events:

| Event | Description |
|-------|------------|
| `vm.created` | VM provisioning started |
| `vm.running` | VM is ready |
| `vm.stopped` | VM has stopped |
| `vm.deleted` | VM was destroyed |
| `vm.resize.started` | Resize in progress |
| `vm.resize.completed` | Resize complete |
