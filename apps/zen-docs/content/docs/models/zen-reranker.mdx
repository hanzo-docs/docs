# zen-reranker

Cross-encoder reranker model for improving search result quality by re-scoring candidate documents.

## Overview

zen-reranker is built on Qwen3-Reranker and fine-tuned for the Zen embedding ecosystem. It takes a query-document pair and outputs a relevance score, enabling more accurate ranking than bi-encoder similarity alone.

Designed as a companion to [zen-embedding](/docs/models/zen-embedding) for two-stage retrieval pipelines.

## Specifications

| Property | Value |
|----------|-------|
| **Base Model** | Qwen3-Reranker |
| **Max Tokens** | 8,192 |
| **License** | Apache 2.0 |
| **Type** | Cross-encoder |

## Quick Start

```python
from sentence_transformers import CrossEncoder

reranker = CrossEncoder("zenlm/zen-reranker")

query = "What is quantum computing?"
documents = [
    "Quantum computing uses quantum mechanics for computation",
    "Classical computers use binary logic gates",
    "How to bake chocolate chip cookies"
]

# Score each query-document pair
scores = reranker.predict([(query, doc) for doc in documents])

# Sort by relevance
ranked = sorted(zip(scores, documents), reverse=True)
for score, doc in ranked:
    print(f"{score:.4f}: {doc}")
```

## Two-Stage Retrieval Pipeline

```
Query → zen-embedding (bi-encoder) → Top-100 candidates → zen-reranker (cross-encoder) → Top-10 results
```

1. **Stage 1**: Use zen-embedding for fast approximate similarity over millions of documents
2. **Stage 2**: Use zen-reranker to re-score the top candidates with higher accuracy
