## The Zen MoDE Architecture

Zen MoDE stands for **Mixture of Distilled Experts**. The core idea is straightforward:

- Each model contains a large set of specialized expert networks
- During inference, only a small subset of experts activates per token (sparse routing)
- Experts are initialized from, and continuously refined against, best-in-class research foundations
- The result is a model that matches the capability of much larger dense models at a fraction of the inference cost

This is not a gimmick. The combination of sparse expert routing and distillation-based initialization lets us achieve frontier capability at model sizes that actually run on hardware people own.

For the full technical treatment, see [Zen MoDE Architecture](/blog/zen-mode-architecture/).

## The Research Lineage

We have been working on this for a while:

- **Zen 1 (2023)**: Initial architecture exploration. Small models, public training logs, proof that the distillation approach worked.
- **Zen 2 (2024)**: First production-quality release. 7B to 70B parameter range. Adopted by Zoo Foundation partners for research use.
- **Zen 3**: Multimodal expansion — vision and audio integrated into the same model family. Embedding and reranker models added.
- **Zen 4 (current)**: Flagship generation. 480B MoE at the top end. 1M token context. Full coverage from 0.6B edge models to 744B research-scale.
- **Zen 5 (in training)**: 1T+ parameter scale, 2M context, tighter vision-text-audio integration.

## On the Abliterated Variants

We release abliterated versions of the major models — variants with safety refusals removed at the weight level. These exist for legitimate research purposes: studying model behavior, evaluating jailbreak robustness, and building custom safety layers from scratch rather than working around ours.

They are clearly labeled. If you need to ask whether you should use them, you probably shouldn't.

## Open Weights Is the Only Credible Commitment

We have watched enough AI companies announce "open" models that ship without weights, or with licenses so restrictive they are effectively closed. We are not doing that.

- Download weights from [huggingface.co/zenlm](https://huggingface.co/zenlm)
- Run locally with `transformers`, `vLLM`, or `llama.cpp`
- Fine-tune on your own data
- Deploy commercially without a royalty agreement

## How to Use Zen

**Try it free:** [hanzo.chat](https://hanzo.chat) — no account required, 14 Zen models in one interface.

**API access:** [console.hanzo.ai](https://console.hanzo.ai) — OpenAI-compatible endpoint, per-token billing, all Zen models available.

**Download weights:** [huggingface.co/zenlm](https://huggingface.co/zenlm) — full catalog, all variants, Apache 2.0.

```bash
# Run with vLLM (recommended for production)
pip install vllm
vllm serve zenlm/zen4-pro --tensor-parallel-size 4

# Run with Transformers (development / smaller models)
pip install transformers torch
python -c "
from transformers import pipeline
pipe = pipeline('text-generation', model='zenlm/zen4-flash')
print(pipe('Explain sparse expert routing in three sentences.')[0]['generated_text'])
"

# Run quantized on a laptop (Zen Nano)
pip install llama-cpp-python
# download zenlm/zen-nano-q4_k_m.gguf from HuggingFace
```

**Use the Hanzo API:**

```bash
curl https://api.hanzo.ai/v1/chat/completions \
  -H "Authorization: Bearer $HANZO_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "zen4-pro",

  }'
```

## Training Is Open Too

Zoo Labs Foundation publishes training infrastructure under the Zoo Gym project:

- GRPO (Group Relative Policy Optimization) implementation
- ASO (Active Semantic Optimization, HIP-002) for continuous improvement from production signals
- BitDelta (ZIP-007) for behavioral compression and efficient weight updates
- DSO (Decentralized Semantic Optimization, ZIP-001) for federated fine-tuning

The full training stack is available on GitHub. We publish training logs, data provenance documentation, and evaluation results — including failures. Reproducibility is not an afterthought.
