## Headline claim

> Every signature produced by a Pulsar threshold ceremony
> (DKG → Round-1 → Round-2 → Combine) is **bit-identical** to a signature
> produced by single-party FIPS 204 ML-DSA-65 on the same message and
> group public key.

A FIPS-validated ML-DSA verifier (BoringSSL FIPS, AWS-LC, OpenSSL 3.0 PQ
provider) accepts a Pulsar signature without modification.

### Two-variant disclosure

The submission ships **both**:

- The **v0.1 reveal-and-aggregate flow** — the runnable Go reference +
  KAT vectors + 19/19 N1 interop subtests.
- The **v0.2 FSwA Lagrange-linearity flow** — the Jasmin sources
  type-check against this; the EasyCrypt N1 reduction is stated against
  this.

Both variants produce a FIPS 204 byte-equal $\sigma = (\tilde{c},
\mathbf{z}, \mathbf{h})$. The v0.1 path carries a documented
reconstruction-aggregator trust caveat; v0.2 avoids the reconstruction
via Lagrange-linearity but is not yet the reference runtime.

The Class N1 byte-equality theorem in EasyCrypt is stated and proved
against the centralized recovery model; both wire variants emit
$\sigma = (\tilde{c}, \mathbf{z}, \mathbf{h})$ byte-equal to single-party
FIPS 204.

### Theorem framing — accepted-path correctness

The Class N1 byte-equality theorem is **conditional on acceptance**:
*if* the threshold combine (and the single-party comparator) accepts —
i.e., passes the ML-DSA norm and rejection checks and the kappa
rejection-sampling loop converged — *then* the produced byte string
equals the centralized ML-DSA-65 signature on the same protocol-level
inputs, under the stated layout, algebraic, and byte-walk assumptions.

ML-DSA rejection sampling remains probabilistic per FIPS 204; the
deterministic EC model captures the accepted-path conditioning via the
`accept_signing_attempt` predicate rather than via probabilistic Hoare
logic.

## What to run

The reproducibility gate is `scripts/build.sh` against the tarball
extract — the entire submission is self-contained, so no network access
is required:

```bash
tar xzf submission-YYYY-MM-DD.tar.gz
cd pulsar
scripts/build.sh          # builds Go ref + spec PDF
scripts/test.sh           # runs unit + KAT + interoperability tests
scripts/bench.sh          # produces signature/verification benchmarks
scripts/gen_vectors.sh    # regenerates KAT vectors (deterministic)
```

`scripts/build.sh` exits non-zero on any failure. CI runs the same
script on every commit; the reproducibility property is the load-bearing
one for the submission.

## Class N4 — public-key preservation across resharing

Multi-party proactive resharing preserves the group public key across
committee rotations (epoch boundaries), so a single long-lived public
identity persists while the secret-share custodians rotate.

artifacts. Drift is a build bug — please open an issue.
