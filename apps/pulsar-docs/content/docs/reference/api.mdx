The reference implementation is a single flat package at
`github.com/luxfi/pulsar/ref/go/pkg/pulsar`. This page covers the
load-bearing exports. Full godoc:
[`pkg.go.dev/github.com/luxfi/pulsar/ref/go/pkg/pulsar`](https://pkg.go.dev/github.com/luxfi/pulsar/ref/go/pkg/pulsar).

**Status — pre-spec-freeze.** The exported API will change up to the spec
encoding freeze (target end of August 2026). After freeze the API is fixed
and a `v0.1` tag is cut.

All cryptographic operations route through:

- `hash`        — NIST SP 800-185 (cSHAKE256 / KMAC256 / TupleHash256) only
- `primitives`  — Module-LWE polynomial-vector arithmetic
- `sign`        — 2-round threshold sign protocol
- `dkg`         — Pedersen DKG over $R_q^k$
- `reshare`     — proactive resharing with beacon-randomized quorum
- `fmt`         — wire encoding / KAT format

All logging — including for any error path — uses `github.com/luxfi/log`.
The standard library's `log` package is **forbidden** in this module:
a secret-correlated value reaching stdout is a side channel even at debug
level (HIP-0077 red review F8). Enforced by a CI lint, not just convention
(`scripts/checks/test/no-secret-logs.sh`).

## Parameter modes

```go
type Mode uint8

const (
    ModeUnspecified Mode = 0
    ModeP44         Mode = 44 // NIST PQ Category 2
    ModeP65         Mode = 65 // NIST PQ Category 3 (production target)
    ModeP87         Mode = 87 // NIST PQ Category 5
)
```

`Pulsar-65` is the production target for Lux consensus (`SigSchemePulsar65`).

```go
type Params struct {
    Mode Mode

    // FIPS 204 module-LWE shape.

    N     int    // 256 — ring degree
```

## Single-party signing

```go
func GenerateKey(params *Params, rng io.Reader) (*PrivateKey, error)
func KeyFromSeed(params *Params, seed [SeedSize]byte) (*PrivateKey, error)

const SeedSize = 32

type PrivateKey struct { /* ... */ }
type PublicKey  struct { /* ... */ }

func (sk *PrivateKey) Public() *PublicKey
func (pk *PublicKey)  Equal(other *PublicKey) bool

func Sign(
    params *Params,
    sk     *PrivateKey,
    message, ctx []byte,
    randomized bool,
    rng        io.Reader,
) (*Signature, error)
```

`Sign` produces a 3309-byte (for Pulsar-65) `*Signature` wire-identical to
`cloudflare/circl`'s `mldsa65.Sign` from the same seed and message.
`randomized=true` (the default for production) uses the FIPS 204 hedged
variant; `randomized=false` is deterministic and KAT-reproducible. If
`rng` is `nil` and `randomized=true`, `crypto/rand.Reader` is used.

Source: `ref/go/pkg/pulsar/sign.go`, `keygen.go`.

## Threshold signing — v0.3 algebraic (recommended)

The algebraic aggregator is **public-only at sign time** — no master
secret reconstruction in aggregator memory. This is the N1 submission
target.

```go
    params  *Params,

func (s *AlgebraicThresholdSigner) Round1() (*AlgebraicRound1Message, error)

    round1 []*AlgebraicRound1Message,

    round1 []*AlgebraicRound1Message,
    peerW  map[NodeID]polyVec,

    setup  *AlgebraicSetup, // public-only material
```

`AlgebraicAggregate` has **no** `*PrivateKey`, **no** `SkBytes`, **no**
`seed` parameter — its public-BFT safety contract is enforced by a Go AST
walk in `TestAlgebraic_NoSkAccess` that fails if any sk-bearing primitive
is reachable from `AlgebraicAggregate`'s body, including indirect dispatch
(hardened in v1.0.22).

Source: `ref/go/pkg/pulsar/threshold_v03.go`.

## Threshold signing — v0.2 transitional (legacy)

The transitional aggregator reconstructs the master ML-DSA secret in
aggregator memory and calls `mldsaSign(setup.SkBytes, ...)`. Output
byte-equality holds, but the TCB at sign time includes the aggregator.
v0.2 is documented for protocol-evolution traceability; the N1 submission
targets v0.3.

```go
type TransitionalSetup struct {
    /* SkBytes — present; declared as v0.2 TCB defect */
    /* ... */
}
type TransitionalThresholdSigner struct { /* ... */ }

func DealTransitionalShares(/* ... */) (*TransitionalSetup, []*PolyKeyShare, error)
func NewTransitionalThresholdSigner(/* ... */) (*TransitionalThresholdSigner, error)

func (s *TransitionalThresholdSigner) Round1() (*TransitionalRound1Message, error)
func (s *TransitionalThresholdSigner) Round2(
    round1 []*TransitionalRound1Message,
) (*TransitionalRound2Message, *AbortEvidence, error)

func TransitionalAggregate(/* SkBytes-dependent */) (*Signature, error)
```

`TestTransitional_DependsOnSkBytes` runs the full Round1/Round2 cycle
then sets `setup.SkBytes = nil` and asserts `TransitionalAggregate`
returns `ErrTransitionalNoSetup`. The test currently PASSES (because v0.2
indeed depends on SkBytes); the original gate expected this to FAIL once
v0.3 shipped — the actual closure shipped v0.3 as a parallel track and
left v0.2 honest. Both paths now coexist.

Source: `ref/go/pkg/pulsar/threshold_v02.go`.

## Errors

Each subsystem declares its own sentinel errors with descriptive names
that match the failure mode:

- `ErrNilKey`, `ErrModeMismatch`, `ErrCtxTooLong` (`sign.go:30`)
- `ErrTransitionalNoSetup`, `ErrTransitionalRound1MACBad`,
  `ErrTransitionalRound2MACBad`, `ErrTransitionalRound2CommitBad`,
  `ErrTransitionalRestart` (`threshold_v02.go`)
- DKG, reshare, identity sentinels in their respective files

`*AbortEvidence` carries signed evidence of an identifiable abort
suitable for slashing.

## Determinism for KAT replay

The KAT regen pipeline (`scripts/gen_vectors.sh`) seeds all randomness
from a 48-byte master seed. Calling `Sign` with `randomized=false` and
the corresponding deterministic reader produces a byte-identical
signature on every architecture. CI runs the regen on every commit and
compares against the committed vectors.
