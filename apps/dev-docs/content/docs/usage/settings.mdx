title: Settings
description: Full-screen TUI settings panel for model, theme, agents, and more.
---

The settings overlay lets you change models, themes, Auto Drive defaults, agents, notifications, and more without leaving the chat.

## Open & Navigate

- `/settings` opens the overview
- `/settings <section>` jumps directly to a section
- `/auto settings` and `/update` route into their respective sections

### Keyboard Controls

| Key | Action |
| --- | --- |
| `↑/↓` or `j/k` | Move selection |
| `Tab/Shift+Tab` | Cycle sections |
| `Home/End` | Jump to list ends |
| `Enter` | Open/activate |
| `Esc` | Close section (twice to close overlay) |
| `?` | Toggle inline help |

## Sections

| Section | Description |
| --- | --- |
| **Model** | Pick default chat model and reasoning effort |
| **Theme** | Choose theme and spinner |
| **Updates** | View upgrade channel and status |
| **Agents** | See built-in/custom agents, enable/disable, force read-only |
| **Prompts** | Edit saved prompt snippets |
| **Auto Drive** | Review/agents/QA toggles, continue mode, model override |
| **Review** | Choose review model, toggle auto-resolve |
| **Planning** | Pick model/effort for planning turns |
| **Validation** | Toggle validation groups and tools |
| **Limits** | Read-only view of rate limits and context usage |
| **Chrome** | Browser connection options |
| **MCP** | Enable/disable MCP servers |
| **Notifications** | Toggle and filter notifications |

## Persistence

Changes write to `~/.hanzo/config.toml` when available. Access mode can be stored per workspace; other settings apply globally unless overridden per project.

## Commands

```bash
/settings [section]
/auto settings
/update
/notifications [on|off|status]
```
