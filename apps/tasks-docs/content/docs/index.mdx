# Hanzo Tasks

**Hanzo Tasks** is a durable workflow execution engine for AI agents. Fork of [Temporal.io](https://temporal.io), rebranded and integrated into the Hanzo ecosystem.

## What is Durable Execution?

Durable execution means your workflows **survive crashes, retries on failure, and maintain full event history**. When a server restarts, workflows pick up exactly where they left off. When an activity fails, it retries automatically with configurable backoff. Every step is recorded for full observability.

## Key Features

- **Durable Workflows** -- survive server crashes and restarts
- **Automatic Retries** -- configurable retry policies with exponential backoff
- **Timeouts** -- enforce deadlines on long-running tasks
- **Event Sourcing** -- full replay history for every execution
- **Priority Queues** -- urgent tasks execute first
- **Fan-Out/Fan-In** -- parallel execution with result aggregation
- **Signals and Queries** -- interact with running workflows
- **Multi-Language** -- Go SDK, Python SDK, REST API

## Quick Start

### Go

```go
import "go.temporal.io/sdk/client"

c, _ := client.Dial(client.Options{
    HostPort: "tasks.hanzo.ai:7233",
    Namespace: "your-org",
})

run, _ := c.ExecuteWorkflow(ctx, client.StartWorkflowOptions{
    ID:        "agent-task-1",
    TaskQueue: "agents",
}, AgentTaskWorkflow, input)

var result AgentResult
run.Get(ctx, &result)
```

### Python

```python
from hanzo_tasks import Client, TasksConfig

client = await Client.connect(TasksConfig(
    address="tasks.hanzo.ai:7233",
    namespace="your-org",
))

handle = await client.submit(agent_task_workflow, input, queue="agents")
result = await handle.result()
```
