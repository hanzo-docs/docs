Every app you use today works the same way: your data lives on someone else's computer, encrypted with someone else's keys, governed by someone else's terms. You rent access to your own information.

We built the replacement.

## The Thesis

**Base Vault SDK is the personal computer of Web5. Lux is the trust kernel.**

Four sentences define the architecture:

- **Base** owns state — encrypted SQLite vault, local-first, offline-capable
- **Lux** owns trust — identity, key policy, threshold governance, receipts
- **Providers** own service quality — relay, storage, compute, bandwidth
- **Users** own keys and exit rights — switch providers without losing data

That last point is the one that matters. Exit rights are the entire game.

## Why Old Crypto Failed

The first generation of crypto apps tried to put everything on-chain: the database, the app server, the compute, the UX. Users got 30-second confirmation times, $50 gas fees, and public state that anyone could read.

That was the wrong abstraction.

Chain is not a database. Chain is not a server. Chain is **trust infrastructure** — the place you go when you need global agreement that two parties cannot provide alone.

## The Right Abstraction

Every user, organization, and AI agent gets a **capsule**:

- An encrypted SQLite vault (AES-256-CBC page encryption via sqlcipher)
- A CRDT sync log for offline-first collaboration
- A DID identity bound to their keys, not to an email/password
- Threshold controls for high-value actions (2-of-3 approval, recovery, reveal)
- An audit chain for receipts that neither party can deny
- A compute policy (local, provider, or confidential via FHE)

Cloud is optional replication. Chain is optional trust anchoring. Both are services you choose, not landlords you depend on.

## Three Classes of Apps

### 1. Local-First Private Apps

The default. A user installs an app and gets a personal vault instantly. Notes, files, messages, credentials, AI memory, preferences — all encrypted, all local, all theirs. No account creation beyond key creation. Cloud is just sync.

This kills the assumption that the server owns truth.

### 2. Threshold Apps

Important actions require quorum: treasury approval, admin recovery, secrets reveal, regulated access, multi-party workflow signoff. This is where threshold cryptography stops being a crypto curiosity and becomes a product primitive.

Your company's admin account isn't a single password. It's 2-of-3 executives attesting. Recovery isn't "email a link." It's 3-of-5 guardians.

### 3. Confidential Compute Apps

The moonshots: private AI inference where the provider never sees your prompt. Sealed-bid auctions where nobody sees the bids until matching. Encrypted analytics where the data owner shares insights without sharing data. Private DeFi where trades execute without front-running.

This is where Fully Homomorphic Encryption (TFHE, CKKS) creates qualitatively new products — not just better versions of existing ones.

## The Stack

We don't pretend one primitive does everything. The stack is layered:

**Default (every app):** AES-256-GCM data encryption + ML-KEM key wrapping. Fast, post-quantum safe, no threshold required. Every secret at rest uses this.

**Threshold control (high-value actions):** TFHE gates threshold key reveal. K-Chain stores key shares; T-Chain decrypts only when the required number of validators attest. For treasury, recovery, admin, regulated access.

**Confidential compute (new product categories):** CKKS scheme for encrypted arithmetic on ML model inputs. Submit encrypted inference requests; evaluate under encryption; decrypt results only for the requester. GPU-accelerated: 40x speedup over CPU for FHE operations.

## AI Agents as Sovereign Processes

This is where it gets interesting.

An AI agent today is an API script — stateless, identity-less, running on someone else's infrastructure with someone else's keys. It has no memory persistence, no privacy from its host, no economic autonomy.

In our architecture, every agent gets its own capsule:
- Encrypted vault for persistent memory
- DID identity for authentication
- Scoped capabilities (read/write/sign/spend)
- Threshold spend controls (T-Chain approval for high-value actions)
- Auditable action history anchored on-chain
- Private inference path via CKKS (provider never sees inputs)

Agents stop being chatbot wrappers and start being sovereign processes.
