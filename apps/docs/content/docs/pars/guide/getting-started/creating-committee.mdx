# Creating a Committee

This guide walks through creating a new Committee, Sub-DAO, or Working Group in Pars DAO.

## Prerequisites

Before creating a Committee:

- [ ] Minimum 10,000 veASHA for proposal submission
- [ ] Clear mandate and mission statement
- [ ] Identified initial members (3-7 recommended)
- [ ] Budget request prepared
- [ ] Success metrics defined

## Committee Types

### PARS-10 Committee

The 10 core Committees are already established. New PARS-10 Committees require a Constitutional Amendment (67% supermajority, 5M veASHA quorum).

### Sub-DAO

Independent project under a Committee with dedicated governance:

- Multi-year mandate
- Own treasury vault
- Choice of governance model
- Recallable funding

### Working Group

Temporary task force with limited scope:

- Maximum 6-month term (renewable)
- Uses parent Committee governance
- Budget cap from Committee allocation
- Single coordinator

## Creating a Sub-DAO

### Step 1: Draft the Proposal

Navigate to your parent Committee and click **"New Sub-DAO"**.

Fill in the required fields:

```yaml
Name: Persian Film Festival DAO
Persian Name: جشنواره فیلم پارسی
Parent Committee: Culture (فرهنگ)

Mandate: |
  Annual film festival celebrating Persian cinema,
  supporting filmmakers, and preserving cinematic heritage.

Duration: 12 months (renewable)

Governance:
  Type: hybrid
  Multisig:
    Threshold: 2
    Signers: 3
  TokenVeto:
    Threshold: 10000  # veASHA to veto
```

### Step 2: Define Funding

Specify your budget request:

```yaml
Funding:
  Initial: $50,000
  Annual: $100,000
  Source: culture_committee_allocation

Disbursement:
  Schedule: quarterly
  Conditions:
    - Milestone completion
    - Impact report submission

Recall Conditions:
  - Missed milestones (30-day grace)
  - Committee vote
  - Performance score below 60
```

### Step 3: Set Milestones

Define clear deliverables:

```yaml
Milestones:
  - id: M1
    date: 2026-03-01
    deliverable: "Venue secured, initial programming announced"
    budget: $15,000

  - id: M2
    date: 2026-06-01
    deliverable: "Film submissions closed, jury selected"
    budget: $10,000

  - id: M3
    date: 2026-09-15
    deliverable: "Festival execution complete"
    budget: $20,000

  - id: M4
    date: 2026-10-15
    deliverable: "Impact report published"
    budget: $5,000
```

### Step 4: Configure Governance

Choose your governance model:

**Multisig (Recommended for new Sub-DAOs)**
```yaml
Governance:
  Type: multisig
  Threshold: 2
  Signers:
    - 0x...  # Coordinator
    - 0x...  # Operations lead
    - 0x...  # Finance lead
```

**Token Voting**
```yaml
Governance:
  Type: token-voting
  Token: veASHA  # or dedicated sub-token
  Quorum: 10000
  Approval: 50
  VotingPeriod: 5 days
```

**Hybrid**
```yaml
Governance:
  Type: hybrid
  Multisig:
    Threshold: 2
    Signers: 3
  TokenVeto:
    Threshold: 5000
    Period: 48 hours
```

### Step 5: Submit for Approval

1. Click **"Preview Resolution"**
2. Review all details
3. Click **"Submit"**
4. Sign the transaction (requires 10K veASHA)

The Resolution enters the voting queue with a 3-day review period.

### Step 6: Voting Period

- 7-day voting period
- Standard Resolution: 50% majority, 1M veASHA quorum
- Promote your proposal in governance forums
- Answer community questions

### Step 7: Execution

After passing:

1. Resolution enters 3-day timelock
2. Treasury Committee deploys vault
3. Initial funding transferred
4. Governance contracts activated
5. Sub-DAO is live!

## Creating a Working Group

Working Groups are simpler to create and don't require a full DAO vote.

### Step 1: Committee Approval

Working Groups only need approval from the parent Committee:

1. Navigate to your Committee
2. Click **"New Working Group"**
3. Fill in details

```yaml
Name: Nowruz Event Planning WG
Coordinator: 0x...
Members:
  - 0x...
  - 0x...
  - 0x...

Duration: 3 months
Budget: $15,000

Deliverables:
  - Event venue secured
  - Programming finalized
  - Logistics coordinated
  - Event executed
  - Report submitted
```

### Step 2: Committee Vote

- Uses Committee's internal governance
- Multisig: Requires threshold signatures
- Token: Committee members vote

### Step 3: Activation

Once approved:
- Budget allocated from Committee funds
- Working Group appears in hierarchy
- Members can access funds per approval levels

## Post-Creation Setup

### 1. Configure Treasury

```
Sub-DAO Dashboard → Treasury → Settings
```

- Add approved signers
- Set spending limits
- Configure streaming payments

### 2. Set Up Reporting

```
Sub-DAO Dashboard → Settings → Reporting
```

Required reports:
- Monthly: Progress update, spending summary
- Quarterly: Full audit, impact metrics
- Final: Comprehensive closeout report

### 3. Establish Processes

Document your operating procedures:

- Decision-making process
- Meeting cadence
- Communication channels
- Escalation paths

### 4. Announce Launch

- Post in governance forum
- Share on community channels
- Update Committee page

## Best Practices

### Do's

- ✅ Start with clear, measurable goals
- ✅ Keep governance simple initially
- ✅ Establish regular reporting habits
- ✅ Maintain open communication
- ✅ Document all decisions

### Don'ts

- ❌ Request more budget than needed
- ❌ Skip milestone planning
- ❌ Ignore parent Committee guidance
- ❌ Delay reporting
- ❌ Operate without transparency

## Troubleshooting

### Resolution Not Passing

- Check quorum requirements
- Engage community in forums
- Address concerns raised
- Consider revising proposal

### Treasury Access Issues

- Verify signer configuration
- Check threshold requirements
- Contact Treasury Committee

### Governance Disputes

- Escalate to parent Committee
- Use Governance Committee dispute resolution
- Document all communications
