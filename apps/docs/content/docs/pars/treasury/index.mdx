# Treasury Management

Pars DAO uses a dual-bucket treasury system: **Working Treasury (WT)** for operational funding and **Endowment (END)** for long-term sustainability. This structure balances immediate operational needs with perpetual preservation.

## Dual-Bucket Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                     Pars DAO Treasury                           │
├───────────────────────────┬─────────────────────────────────────┤
│    Working Treasury       │           Endowment                 │
│         (WT)              │            (END)                    │
├───────────────────────────┼─────────────────────────────────────┤
│ • 24-month runway         │ • 60% of total (locked)             │
│ • Liquid, operational     │ • Long-horizon reserve              │
│ • Allocated to Committees │ • Strict withdrawal rules           │
│ • Monthly disbursements   │ • Yield reinvested                  │
└───────────────────────────┴─────────────────────────────────────┘
```

## Working Treasury (WT)

### Purpose
Fund day-to-day operations of the 10 Committees with a 24-month runway.

### Allocation

| Committee | Allocation | Monthly (~$100K WT) |
|-----------|------------|---------------------|
| Security | 8% | $8,000 |
| Treasury | 12% | $12,000 |
| Governance | 4% | $4,000 |
| Health | 20% | $20,000 |
| Culture | 10% | $10,000 |
| Research | 8% | $8,000 |
| Infrastructure | 12% | $12,000 |
| Consular | 10% | $10,000 |
| Venture | 6% | $6,000 |
| Impact | 10% | $10,000 |

### Disbursement Rules

1. **Monthly release**: 1/24 of Committee allocation released monthly
2. **Rollover**: Unused funds roll to next month (max 3x monthly)
3. **Reallocation**: Unused >3x triggers reallocation vote
4. **Emergency**: Health Committee can request emergency draw

## Endowment (END)

### Purpose
Long-term reserve ensuring perpetual operations. Designed for institutional-grade governance.

### Rules

| Parameter | Value | Rationale |
|-----------|-------|-----------|
| **Lock %** | 60% of total | Sustainability |
| **Max quarterly deploy** | 5% | Preserve principal |
| **Large deploy threshold** | 2% | Requires full DAO vote |
| **Timelocks** | 3-30 days | Based on amount |

### Risk Buckets

```
┌─────────────────────────────────────────────────────────────────┐
│                    Endowment Allocation                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ████████████████████████░░░░░░░░░░░░  Cash/Stables (40%)       │
│  ████████████████░░░░░░░░░░░░░░░░░░░░  Blue-chip (30%)          │
│  ██████████░░░░░░░░░░░░░░░░░░░░░░░░░░  Strategic (20%)          │
│  ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  Venture (10%)            │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

| Bucket | Target % | Assets | Risk Level |
|--------|----------|--------|------------|
| **Cash** | 40% | USDC, DAI, T-Bills | Minimal |
| **Blue-chip** | 30% | LETH, LBTC, stablecoins | Low |
| **Strategic** | 20% | LPARS, protocol tokens | Medium |
| **Venture** | 10% | Early-stage investments | High |

### Deployment Process

```typescript
interface EndowmentDeployment {
  amount: bigint;
  bucket: 'cash' | 'blue-chip' | 'strategic' | 'venture';
  strategy: Address;
  expectedReturn: number;
  riskAssessment: string;
  conflictDisclosure: boolean;
}
```

1. **Proposal**: Venture Committee proposes deployment
2. **Risk Review**: Security Committee reviews risk
3. **Conflict Check**: All signers disclose conflicts
4. **Vote**: If >2% of END, full DAO vote required
5. **Timelock**: 7-30 days based on amount
6. **Execution**: Treasury Committee executes
7. **Monitoring**: Impact Committee tracks performance

## Transparency

### On-Chain Receipts

Every transaction generates an on-chain receipt:

```solidity
struct Receipt {
    bytes32 id;
    address committee;
    address recipient;
    uint256 amount;
    string category;      // "grant", "operational", "investment"
    string description;   // IPFS hash of detailed description
    uint256 timestamp;
}
```

### Public Reports

| Report | Frequency | Contents |
|--------|-----------|----------|
| **Budget vs Actual** | Monthly | Spending by Committee |
| **Treasury Statement** | Monthly | All holdings, inflows/outflows |
| **Endowment Report** | Quarterly | Performance, allocation, risks |
| **Audit Report** | Annual | Third-party financial audit |

### Dashboard

Real-time treasury dashboard at [pars.vote/treasury](https://pars.vote/treasury):

- Total value locked (TVL)
- WT/END breakdown
- Committee spending
- Investment performance
- Upcoming disbursements

## Emergency Procedures

### Emergency Fund
5% of WT reserved for emergencies:
- Security incidents
- Humanitarian crises
- Market dislocations

### Emergency Withdrawal
Guardian Council can authorize emergency withdrawals:

```solidity
function emergencyWithdraw(
    address recipient,
    uint256 amount,
    string memory reason
) external onlyGuardian {
    require(amount <= emergencyReserve, "Exceeds reserve");
    // Execute and log
}
```

### Circuit Breakers

| Trigger | Action |
|---------|--------|
| TVL drop >30% in 24h | Pause all non-emergency spending |
| Backing ratio \<0.9 | Pause ASHA minting |
| Smart contract exploit | Guardian emergency pause |
