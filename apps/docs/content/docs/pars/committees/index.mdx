# PARS-10 Committees

Pars DAO operates through 10 specialized Committees (کمیته‌ها), each with dedicated funding, governance, and accountability. Committees are the operational backbone of the DAO, managing specific domains while maintaining coordination through shared governance.

## Committee Details

### 1. Security Committee (کمیته امنیّت)

**Mandate**: Coercion resistance, private governance, threshold custody, threat-mode operations.

**Owns**:
- Privacy modules (ZK infrastructure)
- Key rotation policy
- Relayers/Paymasters
- Incident response process

**Outputs**:
- Quarterly posture reports
- Incident logs (sanitized)
- Audit plans and results

---

### 2. Treasury Committee (کمیته خزانه)

**Mandate**: Vaults, accounting, audits, execution, receipts management.

**Owns**:
- Timelock + execution router
- Vault factory
- Reporting schema

**Outputs**:
- Budget vs actual (monthly)
- Receipts ledger (public)
- Monthly disclosures

**Special Role**: Endowment steward (manages END bucket)

---

### 3. Governance Committee (کمیته داد)

**Mandate**: Charter, rules, disputes, constraints, constitutional evolution.

**Owns**:
- Charter registry
- Proposal standards
- Dispute resolution process

**Outputs**:
- Charter updates
- Governance research
- Dispute resolutions

---

### 4. Health Committee (کمیته سلامت)

**Mandate**: Humanitarian aid and health programs with privacy for beneficiaries.

**Owns**:
- Emergency pool (rapid response)
- Operator registry (vetted partners)
- Beneficiary privacy system

**Outputs**:
- Verified delivery confirmations
- Impact scorecards
- Partner reports

**Note**: Largest allocation (20%) - serves as primary legitimacy driver.

**Mandate**: Endowment-style investing for sustainability and capacity building.

**Owns**:

**Outputs**:

**Owns**:

**Outputs**:
- State of the DAO (monthly)
- Integrity ratings
- ESG compliance pack

## Committee Governance

### Governance Types

| Type | Description | Committees |
|------|-------------|------------|
| **Multisig** | 3/5 or 5/7 threshold | Security, Consular |
| **Token Voting** | veASHA weighted | Health, Culture, Research, Impact |
| **Hybrid** | Multisig + token veto | Treasury, Infrastructure, Venture |
| **Guardian** | Supreme Council oversight | Governance |

### Joining a Committee

1. **Stake requirement**: Minimum 1,000 veASHA
2. **Application**: Submit via governance forum
3. **Review**: Committee reviews application
4. **Vote**: Existing members vote (or auto-admit if slots available)
5. **Onboarding**: 30-day probation period

## Cross-Committee Coordination

### Joint Resolutions

Committees can propose joint resolutions requiring approval from multiple Committees:

```typescript
interface JointResolution {
  title: string;
  committees: CommitteeType[];  // Required approvers
  quorumPerCommittee: number;   // veASHA threshold per committee
  description: string;
  actions: ProposalAction[];
}
```

### Inter-Committee Transfers

Committees can transfer funds between each other:

1. **Proposal**: Source committee proposes transfer
2. **Acceptance**: Target committee accepts
3. **Execution**: Treasury Committee executes
4. **Logging**: Impact Committee logs transfer

## Bootstrap Phase

During the first 12 months, Committees operate with training wheels:

- **Guardian oversight**: Council can veto harmful actions
- **Reduced thresholds**: Lower quorums for decisions
- **Mandatory reports**: Weekly status updates required
- **Mentorship**: Experienced advisors assigned

### Bootstrap Milestones

Each Committee must achieve minimum viable capabilities:

```
✓ 1 vault deployed
✓ 1 proposal type operational
✓ 1 reporting template in use
✓ Operator registry populated
✓ First deliverable completed with receipts
```
