# Sub-DAOs & Working Groups

Pars DAO Committees can spawn Sub-DAOs and Working Groups to handle specific initiatives. These child organizations receive delegated funding with built-in accountability mechanisms, including fund recall capabilities.

## Hierarchy

```
Pars DAO (veASHA)
    │
    ├── Committee (e.g., Culture Committee)
    │       │
    │       ├── Sub-DAO (e.g., Nowruz Festival DAO)
    │       │       └── Multi-year initiative, own governance
    │       │
    │       └── Working Group (e.g., Film Grant WG)
    │               └── Time-limited task force
    │
    └── Committee (e.g., Research Committee)
            │
            └── Sub-DAO (e.g., Persian Language Institute)
```

## Sub-DAOs

Sub-DAOs are **independent projects** that use Pars DAO governance infrastructure but have dedicated scope and multi-year mandates.

### Creating a Sub-DAO

1. **Proposal**: Committee submits Sub-DAO creation proposal
2. **Requirements**:
   - Clear mandate and success metrics
   - Initial funding request (from Committee allocation)
   - Governance structure (can be simplified)
   - Sunset conditions or renewal process
3. **Approval**: Standard resolution (50% majority, 1M quorum)
4. **Deployment**: Treasury Committee deploys vault and governance contracts

### Sub-DAO Governance

Sub-DAOs can choose from governance templates:

| Template | Description | Use Case |
|----------|-------------|----------|
| **Multisig** | 3/5 or 5/7 signer threshold | Small teams, fast execution |
| **Token Vote** | Dedicated sub-token | Large community initiatives |
| **Delegated** | Parent Committee controls | Early-stage projects |
| **Hybrid** | Multisig + token veto | Balanced approach |

### Fund Recall

**Critical**: Parent Committee retains ability to recall funds from Sub-DAOs.

```solidity
interface ISubDAOVault {
    // Parent Committee can recall unused funds
    function recall(uint256 amount) external onlyParentCommittee;

    // Emergency recall (Guardian Council)
    function emergencyRecall() external onlyGuardian;

    // Scheduled recall (after milestone miss)
    function scheduledRecall(bytes32 milestoneId) external;
}
```

**Recall triggers**:
- Missed milestones (automatic after 30-day grace)
- Committee vote (standard resolution)
- Emergency (security incident, fraud detection)
- Sunset (end of mandate period)

## Working Groups

Working Groups are **temporary task forces** with limited scope and duration.

### Creating a Working Group

1. **Committee Decision**: Single Committee can create WG internally
2. **Requirements**:
   - Defined deliverables
   - Timeline (max 6 months, renewable)
   - Budget cap (from Committee allocation)
   - Lead coordinator assignment
3. **Approval**: Committee multisig (no full DAO vote needed)

### Working Group Structure

```typescript
interface WorkingGroup {
  id: string;
  name: string;
  parentCommittee: CommitteeType;
  lead: Address;
  members: Address[];

  budget: {
    allocated: bigint;
    spent: bigint;
    remaining: bigint;
  };

  timeline: {
    start: Date;
    end: Date;
    milestones: Milestone[];
  };

  status: 'active' | 'completed' | 'dissolved' | 'recalled';
}
```

### Working Group Lifecycle

```
Proposal → Approval → Active → [Milestone Check] → Complete/Renew/Dissolve
                         │
                         └── Recall (if underperforming)
```

## Fund Flow

### Allocation Path

```
Pars DAO Treasury
        │
        ▼ (Budget Resolution)
Committee Vault
        │
        ├──▶ Sub-DAO Vault (via proposal)
        │         │
        │         └──▶ Operational expenses
        │
        └──▶ Working Group (via internal approval)
                  │
                  └──▶ Deliverable-linked payments
```

### Recall Path

```
Sub-DAO/WG Vault
        │
        ▼ (Recall trigger)
Committee Vault
        │
        ▼ (If Committee dissolved)
Pars DAO Treasury
```

## Accountability

### Reporting Requirements

| Entity | Report Type | Frequency |
|--------|-------------|-----------|
| Sub-DAO | Full audit + metrics | Quarterly |
| Working Group | Progress update | Monthly |
| Both | Receipts ledger | Real-time (on-chain) |

### Performance Scoring

Sub-DAOs and WGs receive performance scores (0-100):

```typescript
interface PerformanceScore {
  deliveryScore: number;      // Milestones met on time
  budgetScore: number;        // Spending efficiency
  impactScore: number;        // Measured outcomes
  governanceScore: number;    // Process compliance

  overall: number;            // Weighted average
}
```

**Score thresholds**:
- **80+**: Eligible for renewal/expansion
- **60-79**: Probation, improvement plan required
- **\<60**: Automatic recall consideration

## Smart Contracts

### SubDAOFactory

```solidity
interface ISubDAOFactory {
    function createSubDAO(
        string memory name,
        address parentCommittee,
        uint256 initialFunding,
        GovernanceTemplate template,
        bytes memory config
    ) external returns (address subDAOVault);

    function createWorkingGroup(
        string memory name,
        address lead,
        uint256 budget,
        uint256 durationDays,
        bytes32[] memory milestoneHashes
    ) external returns (bytes32 workingGroupId);
}
```

### RecallController

```solidity
interface IRecallController {
    // Initiate recall vote
    function initiateRecall(
        address target,
        uint256 amount,
        string memory reason
    ) external returns (bytes32 recallId);

    // Execute recall after timelock
    function executeRecall(bytes32 recallId) external;

    // Emergency recall (Guardian only)
    function emergencyRecall(address target) external onlyGuardian;

    // Check if recall conditions are met
    function canRecall(address target) external view returns (bool, string memory reason);
}
```

## Examples

### Example: Creating a Film Festival Sub-DAO

```typescript
const proposal = {
  type: 'CREATE_SUB_DAO',
  name: 'Persian Film Festival DAO',
  parentCommittee: 'culture',
  mandate: 'Annual film festival celebrating Persian cinema',

  funding: {
    initial: parseEther('50000'),  // $50K initial
    annual: parseEther('100000'),  // $100K/year ongoing
    source: 'culture_committee_allocation'
  },

  governance: {
    template: 'hybrid',
    multisig: ['0x...', '0x...', '0x...'],  // 2/3 threshold
    vetoQuorum: 10000  // 10K veASHA can veto
  },

  milestones: [
    { date: '2026-06-01', deliverable: 'Venue secured, initial programming' },
    { date: '2026-09-01', deliverable: 'Festival execution' },
    { date: '2026-10-01', deliverable: 'Impact report published' }
  ],

  sunset: {
    renewalRequired: true,
    renewalPeriod: 'annual',
    recallOnMiss: true
  }
};
```

### Example: Recalling Funds

```typescript
// Committee detects underperformance
const recallTx = await recallController.initiateRecall(
  subDAOVault.address,
  parseEther('30000'),  // Recall $30K of unused funds
  'Milestone 2 missed by 60 days, no communication'
);

// After 7-day timelock
await recallController.executeRecall(recallTx.recallId);

// Funds returned to Culture Committee vault
```
