title: Data initialization
description: Initialize or migrate Hanzo IAM data using JSON config files.
---

When shipping Hanzo IAM as part of a larger product, preload organizations, applications, users, and other data so users get a working setup without manual configuration. Data initialization uses a JSON file that you provide or generate.

This page describes how to **import** and **export** configuration data.

## Import

By default, Hanzo IAM looks for `init_data.json` in the project root at startup and loads it if present. To use a different path, set `initDataFile` in `conf/app.conf`:

```ini
initDataFile = /path/to/your/init_data.json

A template is available at [init_data.json.template](https://github.com/iam/iam/blob/master/init_data.json.template). Copy and rename it to `init_data.json` and customize as needed.

### Docker

Mount the file into the container with a volume:

```bash
docker run ... -v /path/to/init_data.json:/init_data.json

### Kubernetes

Store the file in a ConfigMap and mount it into the Hanzo IAM pod:

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: iam-init-data
data:
  init_data.json:

Mount the ConfigMap in your Deployment, for example:

```yaml
apiVersion: apps/v1
kind: Deployment
...
spec:
  template:
    ...
    spec:
      containers:
      ...
        volumeMounts:
        - mountPath: /init_data.json
          name: iam-init-data-volume
          subPath: init_data.json
      volumes:
      - configMap:
          name: iam-init-data
        name: iam-init-data-volume

## Export

Export all Hanzo IAM config data to a JSON file for backup or migration.

### Using the binary (recommended)

Run Hanzo IAM with the `-export` flag to dump the database to JSON:

```bash
# Export to default location (init_data_dump.json)
./iam -export

# Export to a custom path
./iam -export -exportPath /path/to/backup.json

Export runs after DB init and then the process exits. It works with binary, Docker, or Kubernetes and does not require the Go toolchain.

### Using Go test (from source)

From the Hanzo IAM source tree:

```bash
go test ./object -v -run TestDumpToFile

This creates `init_data_dump.json` in that directory.

### Migrating to another instance

Rename `init_data_dump.json` to `init_data.json`, put it in the root of the target Hanzo IAM instance, and start Hanzo IAM; the data will be loaded automatically.
