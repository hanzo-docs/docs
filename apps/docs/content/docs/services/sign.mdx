# Hanzo Sign

Hanzo Sign (also called Hanzo eSign) is an open-source document signing platform — an AGPLv3 alternative to DocuSign — for creating, sending, and managing legally binding electronic signatures. It runs as a standalone Next.js/React Router application at `esign.hanzo.ai`, integrates with Hanzo IAM for SSO, and can be self-hosted.

Documents are signed cryptographically: the completed PDF is sealed with a signing certificate (a P12 certificate or Google Cloud HSM key), and every action is captured in an audit trail.

## Features

- **PDF Signing**: Upload and sign PDF documents with legally binding, cryptographically sealed signatures
- **Envelopes**: Group multiple documents and attachments into a single signing transaction
- **Templates**: Create reusable document templates with pre-defined fields and bulk send
- **Signing Fields**: Signature, initials, name, email, date, text, number, radio, checkbox, and dropdown fields
- **Recipients & Signing Order**: Assign roles (signer, CC, viewer) and sequential signing workflows
- **Direct Links**: Tokenized public signing links, no account required for signers
- **Folders & Audit Trail**: Organize documents; every action recorded for compliance
- **Teams & Organizations**: Multi-tenant workspaces with role-based access and invites
- **Webhooks**: Real-time notifications for document lifecycle events
- **Per-Org OIDC**: Organization-scoped SSO portals
- **Hanzo IAM SSO**: Single sign-on via Hanzo identity

## Quick Start

### 1. Create a Document

Upload a PDF and add recipients:

```typescript
const response = await fetch('https://esign.hanzo.ai/api/v1/documents', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${apiToken}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title: 'Service Agreement',
    recipients: [
      { email: 'signer@example.com', name: 'Jane Doe', role: 'SIGNER' },
      { email: 'cc@example.com', name: 'John Doe', role: 'CC' }
    ]
  })
})
```

### 2. Add Signing Fields

```typescript
await fetch(`https://esign.hanzo.ai/api/v1/documents/${docId}/fields`, {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${apiToken}` },
  body: JSON.stringify({
    fields: [
      { type: 'SIGNATURE', page: 1, x: 100, y: 500, width: 200, height: 50 },
      { type: 'DATE', page: 1, x: 350, y: 500, width: 150, height: 30 },
      { type: 'TEXT', page: 1, x: 100, y: 560, width: 300, height: 30 }
    ]
  })
})
```

### 3. Send for Signing

```typescript
await fetch(`https://esign.hanzo.ai/api/v1/documents/${docId}/send`, {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${apiToken}` }
})
```

## Field Types

| Type | Description |
|------|-------------|
| `SIGNATURE` | Electronic signature |
| `FREE_SIGNATURE` | Free-form drawn signature |
| `INITIALS` | Signer initials |
| `NAME` | Signer name |
| `EMAIL` | Signer email |
| `DATE` | Date picker |
| `TEXT` | Free text input |
| `NUMBER` | Numeric input |
| `RADIO` | Radio button group |
| `CHECKBOX` | Boolean checkbox |
| `DROPDOWN` | Select from options |

## Webhooks

### Configure Webhook

```typescript
await fetch('https://esign.hanzo.ai/api/v1/webhooks', {
  method: 'POST',
  headers: { 'Authorization': `Bearer ${apiToken}` },
  body: JSON.stringify({
    url: 'https://myapp.com/webhooks/sign',
    events: [
      'DOCUMENT_SENT',
      'DOCUMENT_SIGNED',
      'DOCUMENT_COMPLETED'
    ]
  })
})
```

### Webhook Events

| Event | Description |
|-------|-------------|
| `DOCUMENT_CREATED` | Document uploaded |
| `DOCUMENT_SENT` | Sent for signing |
| `DOCUMENT_OPENED` | Recipient opened document |
| `DOCUMENT_SIGNED` | A recipient signed |
| `DOCUMENT_COMPLETED` | All recipients signed |
| `DOCUMENT_REJECTED` | Recipient rejected |

## Authentication

Hanzo Sign supports multiple authentication methods:

- **Hanzo IAM**: SSO via `https://hanzo.id` (recommended)
- **Google OAuth**: Sign in with Google
- **Microsoft OAuth**: Sign in with Microsoft / Azure AD
- **Generic OIDC**: Any OpenID Connect provider
- **Email & Password**: Native accounts (signup can be disabled for closed deployments)
- **Passkeys**: WebAuthn/FIDO2 authentication, plus optional 2FA

### IAM Integration

```bash
# Environment variables
IAM_URL=https://hanzo.id
IAM_CLIENT_ID=app-sign
IAM_CLIENT_SECRET=<from-kms>
```

## Self-Hosting

### Docker

```bash
docker run -d \
  --name hanzo-sign \
  -p 3000:3000 \
  -e DATABASE_URL=postgres://... \
  -e IAM_URL=https://hanzo.id \
  -e IAM_CLIENT_ID=app-sign \
  ghcr.io/hanzoai/esign:latest
```

### Environment Variables

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string |
| `NEXT_PUBLIC_WEBAPP_URL` | Public URL of the application |
| `IAM_URL` | Hanzo IAM server URL |
| `IAM_CLIENT_ID` | IAM application client ID |
| `IAM_CLIENT_SECRET` | IAM application client secret |
| `NEXT_PRIVATE_ENCRYPTION_KEY` | Document encryption key |
| `NEXT_PRIVATE_ENCRYPTION_SECONDARY_KEY` | Secondary encryption key |
| `NEXT_PRIVATE_SMTP_HOST` | SMTP server host |
| `NEXT_PRIVATE_SMTP_PORT` | SMTP server port |

## API Reference

Hanzo Sign exposes two API surfaces:

- **`/api/v1`** — the stable REST API (contract-based), authenticated with a Bearer API token. The endpoints below are on this surface.
- **`/api/v2`** and **`/api/v2-beta`** — the current tRPC API with an auto-generated OpenAPI schema at `/api/v2/openapi.json`. This surface is evolving and covers newer capabilities (envelopes, folders, embedding).

### Documents

- `POST /api/v1/documents` - Create document
- `GET /api/v1/documents` - List documents
- `GET /api/v1/documents/:id` - Get document
- `POST /api/v1/documents/:id/send` - Send for signing
- `DELETE /api/v1/documents/:id` - Delete document

### Templates

- `POST /api/v1/templates` - Create template
- `GET /api/v1/templates` - List templates
- `POST /api/v1/templates/:id/use` - Create document from template

### Recipients

- `POST /api/v1/documents/:id/recipients` - Add recipients
- `DELETE /api/v1/documents/:id/recipients/:recipientId` - Remove recipient
