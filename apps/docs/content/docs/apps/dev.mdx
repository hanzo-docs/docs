# Dev — GitHub & GitLab

Automated code review, issue and MR triage, and on-demand help by mention — right in your repositories.

## GitHub App

Installs on your org or repos and adds:

- **PR review** — Hanzo reviews pull requests and leaves inline comments.
- **Issue triage** — labels, summarizes, and routes new issues.
- **`@hanzo` mentions** — mention `@hanzo` in a PR or issue to ask a question, request changes, or kick off an agent.

**Install** — from the [console](https://console.hanzo.ai), choose **GitHub**, install the Hanzo GitHub App, and select the org or repositories to grant it. Approve the requested permissions and Hanzo starts reviewing new PRs and issues.

## GitLab

The same review and triage flow for GitLab merge requests and issues.

- **MR review** — inline review comments on merge requests.
- **Issue triage** — summarize, label, and route issues.
- **`@hanzo` mentions** — mention `@hanzo` in an MR or issue to invoke the agent.

**Connect** — from the [console](https://console.hanzo.ai), choose **GitLab**, authorize over OAuth (GitLab.com or self-managed), and pick the groups or projects to enable.

> **One identity, everywhere.** Sign in with Hanzo ([IAM](/docs/iam) OAuth) or paste a `hk-` [API key](/docs/api-keys). The same credential works in every Hanzo surface — see [Integrations & Extensions](/docs/apps).
